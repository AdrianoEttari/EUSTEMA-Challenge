* Ci teniamo a precisare che per ogni test set abbiamo fatto una doppia previsione. Infatti nei dataset test_s e test_od compaiono dei valori nulli. Per tale motivo una volta abbiamo eliminato i valori nulli e abbiamo calcolato le previsioni (prediction_settlement_2.csv); una seconda volta, invece, abbiamo fatto un imputazione e poi le previsioni (prediction_settlement.csv).
* Nel requirements.txt file non è presente il pacchetto tensorflow poiché creava problemi. Una volta creato un vostro environment utilizzando i pacchetti presenti nel file requirements.txt vi basterà inserire sul vostro terminale ‘conda install tensorflow’. 
* In DATA potete trovare due dataset che verranno usati nei notebook inerenti ai modelli (abbiamo diviso il notebook di preprocessing da quello del modello sia per train_s che per train_od). Se non volete runnare tutto il preprocessing e volete passare direttamente ai modelli, potete usare questi.
* Alcuni modelli richiedono ore per runnare, di conseguenza provvediamo anche a inserire una cartella con i pesi dei modelli (models).
